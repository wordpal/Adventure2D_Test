# 2DAdventure 玩家说明（PlayerReadme）

> 本文档面向玩家，说明基础按键、玩法目标与主要敌人特性。

---

## 1. 基础操作（键鼠）

以下内容基于 `PlayerController`（Unity Input System）在项目中的使用方式整理。

- **移动**
  - A / D 或 方向键：左右移动
- **跳跃**
  - Space：跳跃
  - 贴墙时可触发蹬墙跳（向反方向弹开）
- **攻击**
  - 鼠标左键 / J（取决于你的 Input Actions 绑定）：普通攻击
- **滑铲**
  - Left Shift / K（取决于你的 Input Actions 绑定）：滑铲
  - 滑铲期间角色会获得短暂无敌，并消耗体力
- **交互/确认**
  - E / Enter（取决于你的 Input Actions 绑定）：与场景交互物互动
- **暂停/设置**
  - Esc：打开/关闭暂停面板

> 注：本项目使用 Input System，具体按键以 `Input Actions` 资源中的绑定为准；如果你改过绑定，请以你当前工程设置为准。

## 2. 玩法目标与系统

- **探索与推进**
  - 在关卡中移动、跳跃、寻找通路。
  - 通过传送点在不同场景间移动。

- **战斗与生存**
  - 被敌人攻击会掉血。
  - 受击后会进入短暂“受伤无敌”状态，避免同一次接触被连续扣血。

- **体力（Power）**
  - 滑铲会消耗体力。
  - 体力会随时间自动恢复。

- **存档/读档**
  - 在存档点与之交互可保存进度。
  - 读档后会恢复位置与部分角色数值，并加载对应场景。

---

## 3. 场景交互物

- **宝箱（Chest）**
  - 与宝箱交互后打开宝箱。
  - 打开宝箱会触发回血效果。

- **传送点（TelePoint）**
  - 与传送点交互会切换到目标场景，并将角色移动到指定位置。

- **存档点（SavePointRock）**
  - 与存档点交互会保存游戏进度。

---

## 4. 敌人特性（简表）

> 敌人整体采用 `Enemy + State` 的状态机结构，常见状态包括 Patrol / Chase / Skill。

- **Bee（蜜蜂）**
  - 空中巡逻：在一定半径内随机移动。
  - 发现玩家后追击并尝试攻击。

- **Boar（野猪）**
  - 地面巡逻：遇到墙或悬崖会转向。
  - 发现玩家后加速追击。

- **Snail（蜗牛）**
  - 地面巡逻：遇到墙或悬崖会停顿后转向。
  - 发现玩家会进入技能状态（有特殊动画/机制）。

- **SlimeKing（史莱姆王 / Boss）**
  - 未发现玩家：像蜗牛一样地面巡逻，遇到悬崖/墙会短暂停顿。
  - 发现玩家：追击玩家。
  - 砸地攻击：靠近玩家后跳起并落地；只在落地瞬间开启攻击判定（攻击 Hitbox 是子物体，短时间启用）。

## 5. 常见提示

- **如果你感觉“被一下打掉很多血”**
  - 这是因为某些攻击判定使用触发器持续检测；受击无敌会限制连续扣血。

- **如果 UI 无法点击/有警告提示 EventSystem 重复**
  - 场景中只能有一个 EventSystem，通常应由 Persistent 场景保留一个，其他场景删除重复对象。
